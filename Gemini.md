# Gemini.md - Gemini Code専用指示

Gemini Codeはフロントエンド実装（Next.js）を担当する。
バックエンドやコアロジックの変更は**許可制**とする。
共通ルールは **RULES.md** を参照すること。

---

## ⛔ 絶対禁止事項（厳守）

### 1. アーチ生成・編集ロジックの変更禁止
- `frontend/src/components/steps/` 内の各キャンバスにおける **数学的な計算、ポイント生成、カーブ補間、スケーリング、初期値定義ロジック** には、いかなる理由があっても一切触れてはならない。
- ロジックの修正が必要な場合は、必ずユーザーに報告し、ユーザー自身が対応する。Gemini Codeはスタイル（見た目）の調整のみを行うこと。

### 2. Core/Backendの変更禁止
以下のディレクトリは、既存のStreamlitアプリおよびコアロジックとして安定稼働しているため、**原則変更禁止**とする。
- `core/` (形状生成ロジック、ラティス構造など)
- `backend/` (APIエンドポイントの実装)
- `ui/` (既存のStreamlitアプリケーション)

---

## 🛠 役割と実装方針

- **Framework**: Next.js (App Router) + React 19
- **UI Styling**: Tailwind CSS v4 + Radix UI (shadcn/ui pattern)
- **Design System**: "Bionic Sole Lab" 独自のスタイル（ダークネイビー基調、フラットデザイン）を厳守する。
- **3D Visualization**: Three.js + React Three Fiber

---

## ✅ 実装プロセス

1. **理解**: ユーザーの要望を理解し、既存のロジックを壊さないよう細心の注意を払う。
2. **報告**: 修正前に、どのスタイルの変更を行うか明確に説明する。**ロジックに関わる部分は一切触れない。**
3. **確認**: 実装後、ダークモードのデザインが損なわれていないか、既存の機能（アーチ編集など）が正常に動くかを厳密に確認する。
